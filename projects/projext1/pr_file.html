There's many requirements initiating development (ideally it should be business process driven):

  • 1st to sketch an idea/design,
  • intitate code blocks and let the code publish* a web-site (for sketching the content Figma is a tool). [*For content to be published, however, it needs a security level and involves other considerations].

In Codecamy's skillpath lessons (my choice), the core fuctionality is provided by JavaScript(JS): the lessons proceed I've used few of the basics of writing JS-code, the purpose of which to bring about basic functionality on a web-page.


JS-code typically gets called from the index.html that's 1st read by browser. Interpreting of code files is based on 'stacks' (eg. NGINX is built on top of the web services). When 1st landing page a browser resolves the URL, then html code opens the page and its code is where the call gets read. It's a tag called 'script that calls JS code (often, in each index.html, the script makes the call to a JS file with a simple 1-line of code).


If the development harnesses multiple Front End development technology to be published from sub pages of web-page site, then
this requires learning coding in the introduction to node.js and how it can be set up locally.


So beside Node, for instance, local test enviroment is made possible by Node. It involves setting up .nvm, a local environment designed for this development purpose.

I've adopted the use of Zed editor, from which a developer accesses all project code-files. It is a visually minimalistic code editor. However, there are alternative and equivalent development tool products available, for instance, to avoid telemetry collection by VS code.
