Tailwind (OpenSource), has a shortcoming: it doesn't come w/ pre-designed UI components eg. Cards / Buttons. Instead, it's a low-level class, that lets You build custom designs. This means that Cards is possible to implement defining Tailwind components and its @layer components {
(for a) .card (And,) @apply "a FlexBox", justified & centered (or so).
In this project, I jumped into ready-made ICONS [downloadable in two file-formats, SVG is recommended]. The Discovery: ICONS can be placed on web-pages, the default values are not quite enough to produce an ICON of desired size.

Through a Trial & 'No-Error'-msg, the Lesson Learnt: style.css needs a style-class-tag-definition added in the file to be able to adjust SVG-ICON's [**as I had experienced, this was the case: custom.css has GlobalStyles that apply to other projects' web-page content as well, so in some design-case scenario, size-adjusts to Tailwind's class-coded values some of which having absolutely no effect at all, to be able to resize the ICON].
